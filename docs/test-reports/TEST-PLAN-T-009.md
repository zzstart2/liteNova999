## 2. 被测功能模块

### 2.1 FallbackRouter（智能路由器）
- 维护同一模型对应的多个服务商列表及优先级
- 根据服务商健康状态自动选择最优服务商
- 请求失败时自动 Fallback 到下一个可用服务商
- 所有服务商不可用时抛出明确异常

### 2.2 CircuitBreaker（熔断器）
- **关闭态 (CLOSED)**：正常转发请求，失败计数
- **打开态 (OPEN)**：拒绝请求，快速失败，启动恢复计时
- **半开态 (HALF_OPEN)**：放行探测请求，成功则关闭熔断，失败则重新打开
- 可配置：失败阈值、恢复超时、半开探测次数

### 2.3 核心策略
| 策略 | 说明 |
|------|------|
| **优先级路由** | 按配置顺序 + 健康状态选择首选服务商 |
| **自动 Fallback** | 首选失败 → 自动尝试下一个 |
| **熔断保护** | 连续失败达阈值 → 熔断该服务商 |
| **自动恢复** | 熔断超时后 → 半开探测 → 成功则恢复 |
| **全部熔断** | 所有服务商熔断 → 抛出 AllProvidersUnavailableError |
