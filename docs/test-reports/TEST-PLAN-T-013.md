## 2. 被测功能模块

### 2.1 PassthroughProxy（透传代理）
- 接收客户端原始请求（厂商原生格式）
- 根据目标厂商路由到对应适配器
- **不做请求/响应格式转换**，原样透传
- 透传 headers（API Key 除外，由服务端注入）
- 支持流式透传（SSE）

### 2.2 透传路由 (PassthroughRouter)
- 根据请求路径/参数识别目标厂商
- 校验厂商是否已注册
- 路径重写：客户端路径 → 厂商实际路径

### 2.3 透传错误处理
- 厂商侧 HTTP 错误原样返回（保留 status_code + body）
- 网络层错误包装为统一格式
- 超时处理
- 请求体非法时的优雅拒绝

### 2.4 核心流程
```
客户端 → PassthroughProxy → 识别厂商 → 注入凭证 → 转发 → 原样返回响应
```
