# 测试方案: PRJ-LITE999-T-008 多服务商健康检查机制

## 1. 测试概述

| 项目 | 内容 |
|------|------|
| **任务编号** | PRJ-LITE999-T-008 |
| **任务名称** | 多服务商健康检查机制 |
| **功能描述** | 服务商健康监控：响应时间、错误率、可用性检测 |
| **测试类型** | 单元测试 + 集成测试 |
| **技术栈** | Python 3 + pytest |
| **测试日期** | 2025-07-16 |

## 2. 被测功能模块

### 2.1 HealthChecker（健康检查器）
- 对多个服务商执行健康检查
- 记录每次检查的响应时间
- 统计错误率
- 判定服务商可用性状态

### 2.2 核心指标
| 指标 | 说明 |
|------|------|
| **响应时间 (response_time)** | 单次健康检查请求的耗时（秒） |
| **错误率 (error_rate)** | 最近 N 次检查中失败的比例 |
| **可用性 (availability)** | 基于错误率阈值判定：healthy / degraded / unhealthy |

## 3. 测试用例设计

|---------|---------|---------|--------|

| UT-002 | 单个服务商健康检查 - 超时 | 标记为失败，响应时间记录为超时阈值 | P0 |

| UT-004 | 单个服务商健康检查 - HTTP 5xx | 标记为失败 | P0 |
| UT-005 | 单个服务商健康检查 - HTTP 4xx | 标记为失败 | P1 |
| UT-006 | 错误率计算 - 全部成功 | error_rate = 0.0 | P0 |
| UT-007 | 错误率计算 - 全部失败 | error_rate = 1.0 | P0 |
| UT-008 | 错误率计算 - 部分失败 | error_rate 在 (0, 1) 范围内正确 | P0 |
| UT-009 | 错误率计算 - 滑动窗口超出 | 仅统计最近 N 次记录 | P1 |
| UT-010 | 可用性判定 - healthy | error_rate < 0.1 → healthy | P0 |
| UT-011 | 可用性判定 - degraded | 0.1 ≤ error_rate < 0.5 → degraded | P0 |
| UT-012 | 可用性判定 - unhealthy | error_rate ≥ 0.5 → unhealthy | P0 |
| UT-013 | 响应时间统计 - 平均值 | 正确计算平均响应时间 | P1 |
| UT-014 | 响应时间统计 - P95 | 正确计算 P95 响应时间 | P2 |
| UT-015 | 无历史数据时查询状态 | 返回 unknown 状态 | P1 |

### 3.2 集成测试

| 用例编号 | 测试场景 | 预期结果 | 优先级 |
|---------|---------|---------|--------|
| IT-001 | 多服务商批量健康检查 | 所有服务商均返回检查结果 | P0 |
| IT-002 | 部分服务商不可用 | 不可用的标记 unhealthy，其余正常 | P0 |

| IT-005 | 连续多轮检查 + 状态变迁 | 服务商从 healthy → unhealthy → healthy 正确变迁 | P0 |

### 3.3 边界/异常测试

| 用例编号 | 测试场景 | 预期结果 | 优先级 |
|---------|---------|---------|--------|
| ET-001 | 零服务商配置 | 返回空报告，不报错 | P1 |

## 4. 测试环境

- Python 3.x + pytest
- 使用 Mock 模拟 HTTP 请求，不依赖真实外部服务
- 测试覆盖率目标: ≥ 90%

## 5. 通过标准

- 所有 P0 用例通过
- P1 用例通过率 ≥ 90%
- 无内存泄漏或未处理异常
- 测试覆盖率 ≥ 90%
